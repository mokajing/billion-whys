> 版本：v1.1.0
> 生效日期：2026 年 6 月 30 日

## 第 1 章 家长同意机制

### 1.1 首次启动家长验证页

**触发时机**：用户首次打开小程序（`onLaunch` 中检测 `wx.getStorageSync('parent_verified')` 不存在或已过期）。

**页面设计**：

```
┌──────────────────────────────────────┐
│     🦊 欢迎来到「十亿个什么与为什么」  │
│                                       │
│   本小程序专为 2-6 岁小朋友设计       │
│   请家长完成以下简单验证              │
│                                       │
│   [ 简单问答验证 ]                    │
│   [ 扫码验证 ]                        │
│                                       │
│   验证后即可陪同孩子使用              │
│   单次 ≤ 15 分钟 / 每日 ≤ 30 分钟     │
└──────────────────────────────────────┘
```

#### 1.1.1 简单问答验证

**3 道面向成人常识的题目**（题库随机抽 3 题，避免被刷）：

1. 一个标准西瓜大约多重？A. 3kg  B. 30kg  C. 300kg
2. 下列哪个是成语？A. 鸡飞狗跳  B. 鸡飞猫跳  C. 鸡飞鸟跳
3. 现行《未成年人保护法》施行年份？A. 2006  B. 2015  C. 2021
4. 一周几天上学（义务教育阶段）？A. 4  B. 5  C. 6
5. 下列哪个不是城市名？A. 拉萨  B. 库尔勒  C. 鸭绿江
6. 孩子发烧多少度算高烧需就医？A. 38.5℃  B. 39.5℃  C. 40.5℃

**判定标准**：3 题全对即通过，错 1 题重新抽题，错 3 次当日锁定（次日重试）。

**技术实现**：
- 题库 JSON 静态嵌入，不联网，避免爬虫；
- 答题过程不记录任何个人标识，仅写入 `parent_verified = true` 与过期时间戳。

#### 1.1.2 扫码验证

**方案**：家长扫描本小程序在微信"青少年模式"中的官方二维码（每 5 分钟刷新一次）。

**实现**：
- 调用 `wx.scanCode`（仅 `qrCode` 类型）；
- 服务端验证签名（HMAC-SHA256，key 每日轮换）；
- 验证通过后写 `parent_verified_until = Date.now() + 7*24*3600*1000`。

### 1.2 "成人科普"页面二次确认

**触发场景**：从儿童主界面进入隐藏的"成人科普"内容（例如关于"流血""做梦""死亡""繁殖"等敏感话题的科学解释）。

**二次确认流程**：

```
步骤 1：长按卡片右上角"📚"图标 3 秒
步骤 2：弹出"以下内容为科学解释，建议家长陪同阅读"
步骤 3：再次完成 1 道成人常识题（4 选 1）
步骤 4：显示倒计时 5 秒 + "我已确认"按钮
步骤 5：进入成人科普页（页眉常驻"家长陪同中"提示）
```

**退出机制**：
- 切到后台超过 30 秒自动退出；
- 关闭小程序后再次进入需重新验证。

### 1.3 家长身份验证技术方案

| 维度 | 方案 |
| --- | --- |
| 验证强度 | 弱验证（成人常识题 + 可选扫码），不强制实名 |
| 凭证存储 | `wx.setStorageSync('parent_verified_until', Date.now()+7d)` |
| 凭证有效期 | 7 天，过期重新验证 |
| 防绕过 | 每次进入敏感页面前再次校验 `parent_verified_until`，已过期则强制跳回验证页 |
| 防自动化 | 题库随机抽题 + 选项随机排序 + 答题时间 ≥ 3 秒 |
| 防共享 | 扫码方案使用一次性 nonce，5 分钟失效 |
| 数据最小化 | 不记录答题内容，仅记录通过/失败布尔值 |

**特别说明**：本方案不强制实名认证，避免收集身份证号等敏感信息。若监管或微信平台后续要求实名，再升级为微信原生实名能力（不收集证件号，仅获取布尔结果）。

## 第 2 章 内容分级

### 2.1 2-6 岁内容标准（"三无"承诺）

**主界面所有内容严格遵守**：

| 维度 | 标准 | 例示 |
| --- | --- | --- |
| 暴力 | 无任何暴力、打斗、伤害、武器画面 | 不出现"打人""流血""武器"等词 |
| 恐怖 | 无恐怖、惊悚、血腥、灵异画面 | 不出现鬼怪、黑夜尖叫、尸体 |
| 性暗示 | 无任何性暗示、裸露、暧昧画面 | 不出现身体私密部位、亲吻特写 |
| 歧视 | 无地域、性别、种族、残疾歧视 | 不出现"笨""蠢"等贬义词 |
| 商业 | 无品牌植入、无竞品对比 | 不出现具体品牌名 |
| 政治 | 无政治敏感、无历史争议 | 不涉及时政、领导人 |

### 2.2 内容三级分层

| 等级 | 名称 | 呈现方式 | 进入门槛 |
| --- | --- | --- | --- |
| L1 | 儿童好奇层 | 主界面默认 | 家长首次验证 |
| L2 | 家长陪同实验层 | 卡片带"👨‍👩‍👧 家长陪同"角标 | 二次确认 |
| L3 | 成人科普层 | 完全隐藏，长按"📚"3 秒 | 第 1.2 节二次确认 |

### 2.3 科学解释（成人科普层）隐藏策略

**隐藏实现**：
1. 素材库 JSON 中 `level: 3` 的条目不进入主界面渲染列表；
2. 仅在搜索框输入精确关键词（如"流血 科学"）并完成二次确认后显示；
3. 页面顶部常驻黄色提示条：「以下为科学解释，仅供家长参考，不适合儿童独立阅读」；
4. 页脚显示"返回儿童版"按钮，颜色高亮。

### 2.4 实验步骤家长陪同提示

所有带 `experiment: true` 的内容卡片：

1. 卡片标题下方显示橙色横条："⚠️ 本实验含小零件 / 水火 / 化学品，请家长全程陪同"；
2. 进入实验步骤页前弹窗确认："我确认有家长在场"；
3. 每一步骤底部显示"家长确认完成 → 下一步"按钮；
4. 步骤涉及剪刀、火、电、化学品时，加红色感叹号图标。

### 2.5 "流血 / 做梦"等敏感题特殊处理

| 主题 | 儿童层回答 | 成人层补充 |
| --- | --- | --- |
| 为什么会流血？ | "身体里的小修理工在帮忙，就像水管破了要修。" | 凝血机制、血小板、伤口护理 |
| 为什么会做梦？ | "睡觉时大脑在放小电影。" | REM 睡眠、神经递质 |
| 人为什么会死？ | "像花一样，开完就会落下。" | 细胞衰老、生命周期 |
| 宝宝从哪里来？ | "妈妈肚子里有颗小种子。" | 受精、胚胎发育 |
| 为什么会下雨？ | "云朵哭了" | 水循环、凝结 |

**强制规则**：
- 敏感主题关键词触发自动降级到 L1 表述；
- L3 表述只在二次确认后展示；
- 所有 L3 表述均经过内部 3 人小组（1 名儿童心理学者 + 1 名家长 + 1 名内容审核员）审核。

## 第 3 章 时间管理

### 3.1 单次使用 ≤ 15 分钟技术实现

**实现要点**：

```javascript
// app.js 全局计时器
App({
  onShow() {
    this.singleUseStart = Date.now();
    this.singleTimer = setInterval(() => {
      const elapsed = Date.now() - this.singleUseStart;
      const remaining = 15 * 60 * 1000 - elapsed;
      if (remaining <= 60 * 1000 && remaining > 0) {
        // 提前 1 分钟温和提示
        wx.showToast({ title: '还有 1 分钟就要休息啦', icon: 'none' });
      }
      if (elapsed >= 15 * 60 * 1000) {
        clearInterval(this.singleTimer);
        this.forceExit('single');
      }
    }, 1000);
  },
  onHide() {
    clearInterval(this.singleTimer);
    // 暂停累计，但不重置；onShow 重新计时
  }
});
```

**关键规则**：
- 切后台 ≥ 5 分钟重新计时；
- 切后台 < 5 分钟继续累计；
- 时间到强制跳转家长验证页，不进入主界面。

### 3.2 每日累计 ≤ 30 分钟技术实现

```javascript
// 每日累计使用时长
const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
let dailyUseMs = wx.getStorageSync(`daily_use_ms_${today}`) || 0;

// 在 onShow 中累加
dailyUseMs += (Date.now() - this.singleUseStart);
wx.setStorageSync(`daily_use_ms_${today}`, dailyUseMs);

if (dailyUseMs >= 30 * 60 * 1000) {
  this.forceExit('daily');
}

// 24:00 自动清零（次日 onLaunch 检测日期变化时清旧 key）
```

**清零策略**：日期变更时检测并删除前一天的 `daily_use_ms_YYYY-MM-DD` 键，避免本地 storage 无限增长。

### 3.3 22:00-次日 6:00 禁用（可选，默认开启）

```javascript
const hour = new Date().getHours();
if (hour >= 22 || hour < 6) {
  if (!wx.getStorageSync('night_mode_disabled')) {
    wx.showModal({
      title: '现在是睡觉时间',
      content: '小朋友该睡觉啦，明天再来玩',
      showCancel: false,
      confirmText: '我知道了',
      success: () => wx.navigateBack()
    });
  }
}
```

家长可在"控制面板 > 时间管理"中关闭此限制，关闭时需重新完成家长验证。

### 3.4 时间到了的友好提示文案

**单次到时**：
> 🌙 你今天学到了好多新知识！
> 眼睛要休息一下，去看看远处吧～
> [ 我去休息了 ]

**每日到时**：
> 🌟 今天的探索结束啦！
> 明天再来发现新的"为什么"。
> 晚安，小朋友～

**夜间禁用**：
> 🌛 月亮出来了，
> 小朋友该睡觉啦～
> 明天太阳公公出来时再见！

**文案设计原则**：
- 不使用"超时""限制""禁止"等冷硬词；
- 不让孩子有"被剥夺感"；
- 给出"明天再来"的积极预期。

## 第 4 章 隐私保护

### 4.1 不收集未成年人个人信息

见《隐私政策》第二章。重申：
- 不收集姓名、身份证、手机号、位置、头像、生物特征；
- 所有使用数据仅存本地 storage；
- 不调用 `wx.getUserInfo` / `wx.getPhoneNumber` / `wx.getLocation` 等敏感 API。

### 4.2 不申请敏感授权

| 授权 | 是否申请 | 替代方案 |
| --- | --- | --- |
| 位置 | 否 | 内容不依赖地理位置 |
| 手机号 | 否 | 不需要登录 |
| 相机 / 相册 | 否 | 不上传儿童照片 |
| 麦克风 | 否 | 不录制儿童语音 |
| 通讯录 | 否 | 不调用社交关系 |
| 蓝牙 | 否 | 不依赖外设 |
| 运动 | 否 | 不读取步数 |

唯一授权：本地存储（`wx.setStorageSync`），必需且不可拒绝。

### 4.3 本地存储数据最小化

**最小化原则**：
1. 仅存储时间管理必需的累计毫秒数；
2. 不存储浏览历史明细（仅存"上次浏览主题"1 个键）；
3. 不存储任何儿童输入内容；
4. 所有键值定期清理（详见《隐私政策》第八章）。

**清理时机**：
- 当日 24:00 清理当日计时键；
- 7 天后清理家长验证状态；
- 90 天后清理举报历史；
- 监护人主动清除可一键全清。

## 第 5 章 广告与商业化限制

> 本章为广告与商业化的总纲，详细广告变现方案见 `ad-monetization-plan.md`。
> 个人主体版本下，本项目采用"免费内容 + 家长侧广告变现"模式，但严格遵守以下儿童内容零广告与未成年人模式零广告两条不可逾越的红线。

### 5.0 两条不可逾越的红线（强制规则）

1. **广告位在未成年人模式开启时不展示**：当 `wx.getChildSceneInfo` 检测到未成年人模式（`force_child_mode = true`）时，全小程序**强制关闭所有广告位**（banner、激励视频、插屏等所有类型）。
2. **广告内容不得出现在 layer1/2/3 等儿童内容页**：L1 儿童好奇层（`/pages/home/*`）、L2 家长陪同实验层（`/pages/experiment/*`）、L3 成人科普层（`/pages/science-adult/*`）严禁接入任何广告组件，严禁调用 `wx.createBannerAd` / `wx.createRewardedVideoAd` / `wx.createInterstitialAd` 等广告 API。

广告位白名单仅限家长侧页面（家长控制面板、内容归档页、关于页、设置页），且须在家长验证通过且非未成年人模式下展示。详见 `ad-monetization-plan.md` 第 3 章与第 4 章。

### 5.1 未成年人内容无广告

1. 主界面（L1 儿童好奇层）**完全无广告**，包括但不限于：
   - 横幅广告（`<ad>` 组件）
   - 激励视频广告（`RewardedVideoAd`）
   - 插屏广告（`InterstitialAd`）
   - 视频/内容广告（`VideoAd`）
2. **L1/L2/L3 全部儿童内容页零广告**：儿童好奇层（`/pages/home/*`）、家长陪同实验层（`/pages/experiment/*`）、成人科普层（`/pages/science-adult/*`）严禁接入任何广告组件与广告 API（详见 5.0 节红线 2）。
3. 不接入任何广告 SDK；
4. 不展示第三方品牌 logo、商品图、电商链接；
5. 不出现"点击领红包""签到得奖励"等诱导行为。

### 5.2 无虚拟支付

1. 不调用 `wx.requestPayment`；
2. 不接入小程序内购（虚拟商品、会员、道具）；
3. 不展示任何价格、付费墙、订阅入口。

### 5.3 无诱导分享 / 关注

1. 不出现"分享到群解锁内容""邀请好友得奖励"等机制；
2. 不调用 `wx.shareAppMessage` 进行强制性分享引导；
3. 不要求关注公众号、视频号；
4. 不设置"集赞""砍价""拼团"等社交裂变机制。

### 5.4 其他商业化限制

| 限制项 | 实现承诺 |
| --- | --- |
| 排行榜 | 无 |
| 竞技 / PK | 无 |
| 抽奖 / 转盘 | 无 |
| 等级 / VIP | 无 |
| 付费内容解锁 | 无 |
| 第三方跳转 | 无（不跳转外部 H5/小程序） |

## 第 6 章 内容举报机制

### 6.1 "举报不当内容"入口设计

**入口位置**：每张内容卡片右下角常驻"🚩 举报"按钮（字号 24px，灰色低对比，避免儿童误触）。

**点击后弹窗**：

```
┌────────────────────────────────┐
│   举报不当内容                 │
│                                │
│ 请选择问题类型：               │
│  ○ 内容不适合儿童              │
│  ○ 含暴力 / 恐怖 / 性暗示      │
│  ○ 含歧视 / 辱骂              │
│  ○ 含广告 / 商业引导          │
│  ○ 其他（请描述）             │
│                                │
│ [ 取消 ]      [ 提交举报 ]    │
│                                │
│ 我们将在 24 小时内处理        │
└────────────────────────────────┘
```

**点击需先完成家长验证**（防止儿童误触），提交时上传内容快照（卡片 ID + 题目文本），不上传任何用户标识。

### 6.2 24 小时响应流程

```
T+0h   监护人提交举报 → 服务器入库（举报 ID、卡片 ID、问题类型、时间戳）
T+0h   自动邮件/微信通知审核值班人员
T+2h   值班人员初审，决定是否临时下架
T+12h  二审，三人小组（儿童心理学者+家长+审核员）表决
T+24h  处理完成，邮件回执监护人
T+7d   内容整改或永久下架
T+90d  举报快照自动删除
```

**响应 SLA**：
- 紧急（暴力 / 性暗示 / 恐怖）：2 小时内临时下架；
- 一般：24 小时内回复；
- 争议：48 小时内召开三人小组评议。

### 6.3 举报处理 SOP

1. **接收**：举报后台自动接收，生成工单号；
2. **分类**：值班人员按问题类型打标签；
3. **临时措施**：紧急类立即下架；
4. **评议**：三人小组 24 小时内评议；
5. **整改**：作者修订 / 永久下架；
6. **回访**：邮件回执监护人处理结果；
7. **复盘**：每月汇总举报数据，反向优化内容审核标准；
8. **归档**：举报快照 90 天后自动删除。

## 第 8 章 微信未成年人模式适配

### 8.1 接入 wx.getChildSceneInfo

**调用时机**：`App.onLaunch` 中调用 `wx.getChildSceneInfo`（需基础库 ≥ 2.41.0）。

```javascript
wx.getChildSceneInfo({
  success(res) {
    const isChildMode = res.scene === 1011 || res.is_child_scene;
    if (isChildMode) {
      // 强制启用最严格配置
      wx.setStorageSync('force_child_mode', true);
    }
  }
});
```

**适配策略**：
- 检测到未成年人模式时，**所有 L2 / L3 内容强制隐藏**；
- 单次使用上限自动降至 10 分钟；
- 每日累计上限自动降至 20 分钟；
- 关闭夜间禁用的开关强制隐藏（夜间必禁用）；
- 不显示"成人科普"入口；
- **全小程序所有广告位强制关闭**（详见第 5.0 节红线 1）：banner、激励视频、插屏等均不展示，已加载的流量主 SDK 不再调用展示接口。

| 配置项 | 普通模式 | 未成年人模式 |

| 每日上限 | 30 分钟 | 20 分钟 |

本项目**本就无任何消费入口**，未成年人模式下：
- 不出现任何商品 / 价格 / 会员入口（本就没有）；
- 不调用 `wx.requestPayment`（本就不调用）；
- 不接入任何支付 SDK（本就不接入）。

无需额外适配，承诺保持无商业化。

## 第 10 章 上线后持续合规

### 10.1 定期全量自审（每月 1 次）

**自审清单**：

1. **内容审核**
   - 全量扫描素材库 JSON，核查是否含 L1 卡片中包含 L3 关键词；
   - 抽样 10% L1 卡片，三人小组复核；
   - 检查敏感题（流血、做梦、死亡等）L1 / L3 隔离是否生效。

2. **隐私自审**
   - 全量代码扫描敏感 API 调用（`getLocation` / `getPhoneNumber` / `getUserInfo` 等）；
   - 检查本地 storage 是否新增键值；
   - 检查是否新增第三方 SDK；
   - 检查是否新增网络请求域名。

3. **时间管理自审**
   - 测试单次 15 分钟、每日 30 分钟触发逻辑；
   - 测试夜间禁用时段；
   - 测试切后台 5 分钟重置逻辑。

4. **商业化自审**
   - 全量代码扫描 `<ad>` / `RewardedVideoAd` / `wx.requestPayment` / `wx.shareAppMessage`；
   - 检查是否有付费墙、会员、内购入口；
   - 检查是否新增广告 SDK。

5. **未成年人模式自审**
   - 在微信未成年人模式下测试小程序行为；
   - 检查 L3 隐藏是否生效；
   - 检查时间上限是否自动降低。

**自审报告**：每月 1 日生成上月自审报告，归档保留 3 年。

### 10.2 用户举报响应（24 小时内）

见第 6 章。

**额外要求**：
- 紧急举报（暴力 / 性暗示 / 恐怖）2 小时内临时下架；
- 一般举报 24 小时内回复；
- 月度汇总举报数据，反向优化内容审核标准；
- 季度向网信办等监管部门报送（如监管要求）。

### 10.3 微信政策变更跟踪

**跟踪机制**：

1. **订阅微信开放平台公告**：每周检查 1 次[小程序运营规范](https://developers.weixin.qq.com/miniprogram/product/)更新；
2. **订阅网信办、工信部公告**：每周检查 1 次；
3. **内部变更评估**：发现政策变更后 7 个工作日内完成影响评估；
4. **快速响应**：
   - 微信平台规则变更：14 天内适配；
   - 法律法规变更：30 天内适配；
   - 紧急监管要求：按监管时限。
5. **版本记录**：本方案每次修订需更新版本号、生效日期、版本历史。

### 10.4 责任与问责

1. 项目负责人为未成年人保护第一责任人；
2. 内容审核负责人为直接责任人；
3. 数据保护负责人负责隐私合规；
4. 自审报告、举报处理记录、合规评估报告均归档保留 3 年；
5. 发生重大合规事件（数据泄露、违规内容、监管处罚）须 24 小时内报告项目负责人并启动应急响应。

### 10.5 持续改进

- 每季度召开 1 次未成年人保护复盘会议；
- 每年聘请第三方机构进行 1 次未成年人保护合规评估；
- 跟踪国内外最佳实践，持续优化本方案。
