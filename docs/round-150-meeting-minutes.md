## 📋 各专家意见

### 一、战略与商业层

#### CEO - 周远见
- 观点: Sprint 77 标记为"部分通过"是诚实的，但我们要面对现实——插画管线已经连续 3 个 Sprint 未能完全闭环。这不是技术问题，是优先级问题。
- 建议: Sprint 78 必须定义"插画管线闭环"的硬标准：body-001 安全审核通过 + 幼儿实测通过 + 端到端渲染验证通过，三项全绿才算闭环。不能再有"部分通过"。
- 优先级: P0

#### COO - 林实干
- 观点: 第149轮 3 个 P0 未完成进入第150轮，这是第 2 次跨 Sprint 遗留。红灯治理原则（#35-1）已触发。Sprint 78 首要任务不是新功能，是把 Sprint 77 的 P0 清掉。
- 建议: Sprint 78 的前 3 天设为"技术债清零窗口"，不安排新需求。Sprint 78 剩余时间只做"插画管线闭环"一件事。
- 优先级: P0

#### CFO - 钱守正
- 观点: 插画管线反复延期正在产生隐性成本——AI 生图 token 消耗持续增长但无产出。Sprint 77 的 3 个 P0 每延迟一个 Sprint，等效烧掉约 2 个 Sprint 的 AI 预算。
- 建议: 产能评估报告（P1 第149轮）必须本周交付。如果单张插画全流程超过 3 天，需要重新评估"全量插画覆盖"策略。
- 优先级: P1

### 二、产品与设计层

#### CPO - 叶用户
- 观点: 妈妈等了三周了，body 圈层插画还没在 App 里看到一张。我们现在有 2075 张 body 插画，但用户看到的只有 textOnly 模式。这是"云交付"不是"真交付"。
- 建议: Sprint 78 最小闭环：body-001 端到端上线（H5 + MP），用户能真实看到。一张上线的插画 > 2000 张本地的文件。
- 优先级: P0

#### CTO - 陈架构
- 观点: preversion hook 修复已验证通过（postversion 已增加 build:mp-data），版本号同步机制已稳定。但 IllustrationImage 组件在真实图片端到端渲染上仍有未知风险——本地测试用 mock 数据，从未用真实 512x512 webp 跑过。
- 建议: Sprint 78 第一优先级：body-001 真实图片端到端渲染（H5 + MP）。如果组件有性能问题，趁早发现比上线后 crash 好。
- 优先级: P0

#### 用户体验总监 - 苏体验
- 观点: 第149轮的"插画视觉权重评审团"还没启动。body-001 的插画质量到底如何？用户看到的第一张插画就是产品形象的"第一印象"。如果 body-001 插画质量中等，不如先用 textOnly 过渡。
- 建议: 评审团在安全审核通过后立即启动，采用"绿黄红"三色评级：绿（直接上线）/ 黄（可上线但有优化项）/ 红（必须重做）。body-001 必须绿级才能上线。
- 优先级: P1

#### 视觉设计总监 - 彩虹姐
- 观点: body-001~045 的插画我都看过一轮了。色调和风格基本一致，但 body-001~003 的 experiment 层画面构图偏复杂——3 岁孩子在手机屏幕上可能看不清细节。
- 建议: 插画视觉评审增加"小屏可读性"维度：在 375px 宽屏幕（iPhone SE）上，关键信息是否可见。experiment 层插画需要缩小到 280px 宽度测试。
- 优先级: P1

#### 首席插画师 - 墨小暖
- 观点: 命名规约已统一为 topicId-layer{N}.webp，去掉了 _seedream 后缀。但 science 层插画（body-011~045 有 science 层）的 prompt 风格与 layer1~3 不一致——science 层偏写实，这是有意为之还是 bug？
- 建议: 统一插画风格指引：science 层与 layer1~3 是否使用同一套视觉风格？需在视觉权重评审团中明确裁决。
- 优先级: P1

### 三、技术研发层

#### 前端小凡
- 观点: IllustrationImage 组件空 src bug 已修复（immediate watch + else 分支），但真实图片加载性能未经测试。body-001 的 4 张插画（layer1~3 + experiment）同时加载，在低端机上可能产生 200ms+ 的渲染延迟。
- 建议: 增加图片懒加载 + 渐进式加载（先显示模糊占位，再加载原图）。performance benchmark 必须覆盖 Android 8 低端机（测试虫虫第149轮 P2 尚未执行）。
- 优先级: P1

#### 后端老稳
- 观点: CDN 域名/DNS 工单已提交，但临时 OSS 域名验证尚未完成。没有 CDN，插画只能走本地静态文件，这意味着每次 App 更新都要重新下载全部插画。
- 建议: CDN 临时域名验证必须本周完成。如果 OSS 工单延迟，先用本地相对路径作为 fallback，不能等到 CDN 就绪才上线。
- 优先级: P0

#### AI小智
- 观点: 生图通道健康度看板（P1 第149轮）尚未建立。我们不知道 seedream 通道的可用率、平均延迟、失败率。如果通道在 Sprint 78 期间挂了，我们没有备用监控。
- 建议: 健康度看板最小可用版本：监控 3 个指标（可用率、平均延迟、失败率），通过 webhook 在 > 10% 失败率时告警。一天内可完成。
- 优先级: P1

### 四、内容与教育层

#### CCO - 文若水
- 观点: body-001~003 的 L1 回答我已审过，文字质量没问题。但需要确认：插画安全审核中的 prompt 逆向审核，是否检查了 prompt 中使用的英文词汇对 3 岁孩子的认知适配性？
- 建议: prompt 逆向审核增加"目标受众认知适配"维度：检查 prompt 中是否使用了孩子不应接触的英文词汇（如 anatomy, skeleton, realistic）——这些词虽然是给 AI 的，但需要确保不会通过任何途径泄露到用户界面。
- 优先级: P1

#### 心理学家 - 周教授
- 观点: 第149轮预告的"情绪适龄性审核"尚未完成。body-001~003 涉及什么主题？如果是"心脏跳动"或"血液流动"，需要特别关注恐惧反应。3 岁孩子对体内器官的认知非常模糊，容易产生"身体里有可怕的东西"的误解。
- 建议: 情绪适龄性审核必须包含：1) 主题是否适合 3 岁首次接触；2) 插画中的器官表达是否卡通化、去恐怖化；3) L1 回答是否包含"保护身体"的正面引导。三项全部通过才放行。
- 优先级: P0

#### 幼教专家 - 王园长
- 观点: 我在班上试了 body-001 的 L1 回答——"我们的心脏像一个小水泵，咚咚咚地把血液送到全身"。3 岁孩子反应：有的拍胸口模仿，有的问"水泵是什么"。结论：比喻有效，但需要配上"用手摸摸胸口"的互动引导。
- 建议: 每个 body 圈层 topic 增加"互动引导"字段：在 L1 回答下方增加一行"试试看：把手放在胸口，感受心跳"这样的一句话。不增加文字量，但大幅提升亲子互动质量。
- 优先级: P1

#### 科学顾问 - 科普陈博士
- 观点: body-001~003 的科学内容我没看到完整版本。但根据第149轮记录，只说了"安全审核"，没说"科学准确性审核"。科学准确性审核和安全审核是独立的——prompt 可以安全但科学上不准确（比如把心脏画成红色圆形而不是心形结构）。
- 建议: 科学准确性审核必须独立于安全审核。body-001 的插画需要确认：心脏的位置、大小比例、颜色是否正确（不是红色的圆形，而是偏左的拳头大小）。
- 优先级: P0

### 五、合规与风控层

#### 法务 - 张律
- 观点: 第149轮我提出的 AI 人脸肖像权风险，目前只停留在"prompt 约束"层面，没有合规检查机制。如果 seedream 生成的插画中无意出现了具象的人脸特征，我们需要一个检测流程。
- 建议: 插画安全审核流程增加"人脸特征检测"步骤：对每张插画进行人脸识别，如有具象人脸特征则标记为"需人工复核"。
- 优先级: P1

#### 内容安全 - 安全李姐
- 观点: 第149轮我对 Sprint 78 期中检查前安全审核未完成给了一票否决预告。现在 Sprint 78 已开始，body-001~003 的安全审核仍然没有完成。我行使一票否决权：Sprint 78 上线前，body-001~003 安全审核必须完成，否则不同意任何插画上线。
- 建议: 安全审核流程标准化：1) prompt 逆向审核（检查是否含恐怖/血腥/敏感词）；2) 情绪适龄性清单（周教授 5 条标准）；3) 人脸特征检测；4) 科学准确性审核（陈博士）。四步全通过才算"安全审核通过"。
- 优先级: P0（一票否决权）
