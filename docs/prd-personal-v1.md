# 「十亿个什么与为什么」产品需求文档 PRD（个人主体版 v1.0）

> 版本：v1.0（个人主体版）
> 生效日期：2026 年 6 月 30 日
> 适用产品：微信小程序「十亿个什么与为什么」
> 目标用户：2-6 岁儿童及其家长
> 主体类型：个人主体（个人身份证注册）
> 关联文档：`category-qualification-personal.md`、`ad-monetization-plan.md`、`minor-protection.md`、`privacy-policy.md`

---

## 1. 产品定位

### 1.1 一句话定位

> 一款面向 2-6 岁儿童好奇心、由家长陪同使用的**完全免费**的微信小程序问答工具。

### 1.2 核心价值主张

- **对孩子**：270 个"为什么"问题 + 配图，即时响应好奇心；
- **对家长**：无需付费、无需注册、无需授权敏感信息，打开即用；
- **对监管**：未成年人保护方案完整落地，儿童内容页零广告、零付费、零诱导。

### 1.3 与企业主体版的差异

| 维度 | 企业主体版（原方案） | 个人主体版（本 PRD） |
| --- | --- | --- |
| 主体类型 | 有限责任公司 | 个人 |
| 类目 | 亲子-母婴亲子 | 工具-效率 |
| 商业化 | 会员、订阅、内容付费、广告 | **仅广告变现**，无任何付费功能 |
| 内容定位 | 亲子内容 | 免费工具 |
| 流量主 | 企业主体开通 | 个人主体开通（UV ≥ 1000 后） |
| 责任承担 | 有限责任 | 无限责任（通过严格合规降低风险） |

## 2. 目标用户与场景

### 2.1 用户画像

- **主要使用者**：2-6 岁儿童，由家长陪同使用；
- **决策者**：儿童家长（25-40 岁，一二线城市，重视早期教育）；
- **触发场景**：
  - 孩子日常生活中提出"为什么"问题（如"为什么天是蓝的？"）；
  - 家长希望快速给出科学且儿童可理解的回答；
  - 家长希望在日常场景中激发孩子好奇心。

### 2.2 核心使用流程

```
家长打开小程序
  → 首次启动家长验证（成人常识题，详见 minor-protection.md 第 1 章）
  → 主界面（L1 儿童好奇层，270 题列表）
  → 选择问题 → 查看儿童层答案 + 配图
  → 可选：进入 L2 家长陪同实验层（二次确认）
  → 可选：长按进入 L3 成人科普层（二次确认 + 家长验证）
  → 单次 ≤ 15 分钟 / 每日 ≤ 30 分钟时间管理自动触发
  → 家长控制面板（家长侧，Phase 2 起接入广告）
```

## 4. 内容架构

### 4.1 270 题素材库

- 题量：270 题（首期），覆盖自然、身体、生活、科学、情感 5 大类；
- 每题三层表述：
  - L1 儿童好奇层：儿童可理解的简短回答 + 配图；
  - L2 家长陪同实验层：可选的家庭小实验步骤 + 配图；
  - L3 成人科普层：完整的科学解释，仅供家长阅读。
- 数据结构：素材库 JSON 静态嵌入小程序包内，不联网拉取。

### 4.2 配图规范

- 风格：扁平卡通，色彩柔和，无文字或仅少量中文；
- 尺寸：1024×1024 PNG；
- 来源：AI 生成（保留提示词记录）或授权素材；
- 审核：无暴力、恐怖、性暗示、政治敏感、品牌植入。

### 4.3 内容审核机制

- 人工审核：3 人小组（儿童心理学者 + 家长 + 内容审核员）；
- 机器审核：敏感词词库（暴力、性、政治、歧视、品牌）；
- 月度全量自审：详见 minor-protection.md 第 10 章。

## 5. 未成年人保护

详见 `minor-protection.md`，本 PRD 仅列要点：

- 家长同意机制：首次启动成人常识题验证，凭证有效期 7 天；
- 内容分级：L1/L2/L3 三层，L3 隐藏需二次确认；
- 时间管理：单次 ≤ 15 分钟、每日 ≤ 30 分钟、22:00-6:00 禁用；
- 隐私保护：不收集敏感信息，仅本地 storage；
- 广告限制：**儿童内容页（L1/L2/L3）零广告**、**未成年人模式全小程序零广告**；
- 举报机制：24 小时响应，紧急类 2 小时临时下架；
- 微信未成年人模式适配：`wx.getChildSceneInfo` 检测后强制最严格配置。

## 6. 商业化模式

### 6.1 总体策略

完全免费内容 + 家长侧广告变现。详见 `ad-monetization-plan.md`。

### 6.2 三阶段路线

| 阶段 | DAU | 广告策略 | 月收入预估 |
| --- | --- | --- | --- |
| Phase 1 | 0-1000 | 无广告 | ¥0 |
| Phase 2 | 1000-10000 | 家长侧 banner | ¥450 - ¥4500 |
| Phase 3 | 10000+ | + 激励视频（家长侧） | ¥4500+ |

### 6.3 红线

- 儿童内容页（L1/L2/L3）永远零广告；
- 未成年人模式永远零广告；
- 永远不做付费功能；
- 永远不做社交裂变。

## 7. 路线图

### 7.1 总体里程碑

```
M1（2026-07-15 前完成）：个人主体注册
  → 个人身份证 + 实名认证
  → 小程序账号注册完成

M2（2026-07-22 前完成）：类目申请
  → 选择"工具-效率"类目
  → 提交类目申请说明
  → 类目审核通过

M3（2026-08-15 前完成）：MVP 开发完成
  → 270 题素材库就绪
  → 配图就绪
  → 家长验证、时间管理、内容分级、举报机制开发完成
  → 隐私政策、未成年人保护方案发布

M4（2026-08-22 前完成）：MVP 提审与上线
  → 通过 wx-miniprogram-audit-team 的 270 题自审
  → 代码审核通过
  → 正式发布上线

M5（2026-09-30 前完成）：流量增长至 UV ≥ 1000
  → 内容口碑传播
  → 微信搜索优化
  → 家长社群运营

M6（2026-10-15 前完成）：广告接入
  → 申请开通流量主
  → 家长控制面板接入 banner 广告
  → 未成年人模式广告强制关闭验证

M7（2027-06-30 前完成）：DAU ≥ 10000
  → Phase 3 激励视频接入
  → 扩展实验包上线
  → 启动企业主体迁移评估
```

### 7.2 关键决策点

- **M2 后**：若工具-效率类目审核被驳回，启动备选类目"工具-信息查询"；
- **M4 后**：若未成年人内容审核反复被驳回，启动企业主体迁移；
- **M6 后**：若广告收入低于预期 50%，复盘广告位布局与流量来源；
- **M7 后**：若 DAU 稳定 ≥ 5000 且广告月收入 ≥ ¥3000，启动企业主体迁移。

## 8. 技术架构（概要）

### 8.1 技术栈

- 前端：微信小程序原生框架（WXML/WXSS/JS）；
- 数据：素材库 JSON 静态嵌入，不依赖服务端；
- 存储：仅 `wx.setStorageSync` 本地存储；
- 网络：本期不联网（除广告 SDK 与举报上报外）；
- 第三方 SDK：仅微信广告 SDK（Phase 2 起）。

### 8.2 关键页面路径

```
/app.js                                    全局计时器、未成年人模式检测
/pages/parent-verify/parent-verify         首次启动家长验证
/pages/home/home                           主界面（L1，零广告）
/pages/experiment/experiment               实验步骤页（L2，零广告）
/pages/science-adult/science-adult         成人科普页（L3，零广告）
/pages/parent-control/parent-control       家长控制面板（Phase 2 起 banner）
/pages/archive/archive                     内容归档页（Phase 2 起 banner）
/pages/about/about                         关于页（Phase 2 起 banner）
/pages/settings/settings                   设置页（Phase 2 起 banner）
/pages/report/report                       举报页（零广告）
/pages/policy/policy                       隐私政策、用户协议（零广告）
```

### 8.3 合规技术保障

- ESLint 自定义规则：禁止在 `/pages/home/`、`/pages/experiment/`、`/pages/science-adult/` 目录引用广告 API；
- CI 扫描脚本：每月 1 次全量扫描广告 API 引用与敏感 API 调用；
- 广告展示前双重校验：未成年人模式 + 家长验证。

## 9. 成功指标

### 9.1 Phase 1（MVP 上线后 30 天）

- 累计 UV ≥ 500；
- 7 日留存率 ≥ 15%；
- 举报数 < 5 件 / 月；
- 提审一次通过率 ≥ 80%。

### 9.2 Phase 2（UV ≥ 1000 后 30 天）

- 累计 UV ≥ 1500；
- 30 日留存率 ≥ 8%；
- 广告月收入 ≥ ¥300；
- 举报中"含广告"占比 < 5%。

### 9.3 Phase 3（DAU ≥ 10000 后 30 天）

- DAU ≥ 10000；
- 广告月收入 ≥ ¥4500；
- 激励视频观看完成率 ≥ 70%；
- 举报率 < 1%。

## 10. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
| --- | --- | --- | --- |
| 个人主体类目审核被驳回 | 中 | 高 | 备选工具-信息查询；必要时迁移企业主体 |

| 流量增长不及预期 | 高 | 中 | 内容口碑 + 微信搜索优化 + 家长社群 |
| 广告收入低于预期 | 高 | 中 | 不依赖广告收入作为唯一动力 |
| 微信政策变更 | 中 | 高 | 14 天内适配，必要时迁移企业主体 |
