# 《十亿个什么与为什么》小程序隐私保护政策

> 生效日期：2026 年 6 月 30 日
> 版本号：v1.0.0
> 适用范围：微信小程序「十亿个什么与为什么」（以下简称"本小程序"）

---

## 一、协议生效日期与更新机制

1. **生效日期**：本政策自 2026 年 6 月 30 日起正式生效。
2. **更新机制**：本政策可能因业务调整、法律法规变化或微信平台规则更新而修订。当本政策发生重大变更时，我们将在小程序首屏显著位置以弹窗形式提前 7 天公示变更内容，并要求监护人重新确认同意。
3. **版本追溯**：历史版本可向开发者邮箱（见"联系方式"章节）申请获取，保留期不少于 3 年。
4. **修订触发条件**包括但不限于：
   - 收集、使用个人信息的目的、方式、范围发生实质性变化；
   - 接入新的第三方 SDK 或新增信息共享对象；
   - 监护人权利实现机制发生变化；
   - 法律法规或监管要求变更。

## 二、信息收集清单

本小程序面向 2-6 岁儿童及其家长，遵循"最小必要"原则，**严格不收集未成年人个人信息**。

### 2.1 不收集的信息类型

| 类别 | 是否收集 | 说明 |
| --- | --- | --- |
| 未成年人真实姓名 | 否 | 不要求填写 |
| 未成年人身份证号 / 出生年月 | 否 | 不要求填写 |
| 未成年人手机号 | 否 | 不申请 `getPhoneNumber` 授权 |
| 未成年人地理位置 | 否 | 不申请 `getLocation` / `chooseLocation` 授权 |
| 未成年人头像 / 照片 | 否 | 不申请相册、相机授权 |
| 未成年人通讯录 / 设备标识 | 否 | 不申请通讯录、IMEI 授权 |
| 未成年人生物识别信息（指纹 / 面部） | 否 | 不接入任何生物识别接口 |
| 未成年人麦克风录音 | 否 | 不申请 `getRecorderManager` 授权 |
| 微信社交关系链 | 否 | 不调用 `wx.getUserInfo` / 不获取好友列表 |

### 2.2 必要的非个人信息（自动采集）

为保障小程序基本运行，仅在本地处理以下数据，不上传服务器：

| 数据项 | 用途 | 存储位置 | 是否上传 |
| --- | --- | --- | --- |
| 设备品牌 / 型号 | UI 适配（屏幕尺寸） | 本地 storage | 否 |
| 屏幕分辨率 | 内容布局 | 本地 storage | 否 |
| 系统语言 | 中英文切换 | 本地 storage | 否 |
| 微信基础库版本 | 兼容性判断 | 本地 storage | 否 |

### 2.3 监护人主动选择上传的信息

下列信息仅在监护人**主动操作**时产生，且默认不上传：

| 数据项 | 触发场景 | 存储位置 | 保留期 |
| --- | --- | --- | --- |
| 家长陪同问答结果 | 首次启动家长验证页 | 本地 storage | 7 天 |
| 单次使用时长 / 日累计时长 | 时间管理 | 本地 storage | 当日 |
| 举报内容快照 | 监护人点击"举报不当内容" | 服务器 | 90 天后自动删除 |
| 家长控制面板设置 | 家长主动进入控制面板 | 本地 storage | 直到家长重置 |

### 2.4 申请的微信授权清单

本小程序仅申请以下 1 项授权，且均为可选：

| 授权 | API | 是否必需 | 用途 | 拒绝影响 |
| --- | --- | --- | --- | --- |
| 本地存储 | `wx.setStorageSync` | 必需 | 存储使用时长、家长设置 | 无法实现时间管理 |

**不申请**：位置、手机号、相机、相册、麦克风、通讯录、蓝牙、运动数据等任何敏感授权。

## 三、信息使用范围

### 3.1 允许的使用方式

1. **内容展示**：将素材库中的"为什么"问题、答案、插图呈现给用户；
2. **基础运行**：依据屏幕尺寸做 UI 适配；
3. **时间管理**：依据本地 storage 中的累计时长触发休息提示；
4. **举报处理**：在监护人主动举报后，将举报内容快照用于人工审核。

### 3.2 严禁的使用方式

本小程序明确**禁止**以下行为：

1. ❌ 不进行用户画像（profiling）；
2. ❌ 不使用任何推荐算法（recommendation algorithm）；
3. ❌ 不基于使用行为做个性化推送；
4. ❌ 不进行 A/B 测试与行为分析埋点；
5. ❌ 不将使用数据用于商业变现、广告竞价、数据交易；
6. ❌ 不进行精准营销或定向投放；
7. ❌ 不向未成年人推送任何商业信息。

### 3.3 自动化决策声明

本小程序不使用任何自动化决策机制对用户产生法律效力或类似重大影响，不进行自动化画像、自动化推荐、自动化定价。

## 四、信息存储与保护

### 4.1 存储位置

| 数据类别 | 存储位置 | 加密 |
| --- | --- | --- |
| 使用时长 / 家长设置 | 微信小程序本地 storage（`wx.setStorageSync`） | 微信平台沙箱隔离 |
| 举报内容快照 | 阿里云 ECS（华东 2，上海）| TLS 传输 + AES-256 静态加密 |
| 监护人邮箱（仅举报回访时主动提供）| 阿里云 RDS | TLS + AES-256 |

### 4.2 存储期限

- 本地 storage：监护人可通过"家长控制面板 > 重置隐私协议同意"一键清除；
- 举报内容快照：90 天后自动删除；
- 监护人邮箱：仅在举报处理期间使用，处理完毕后 30 天内删除。

### 4.3 安全措施

1. **传输安全**：所有网络请求强制 HTTPS / TLS 1.2+；
2. **访问控制**：服务器仅授权 2 名内部审核人员访问举报数据，操作日志保留 6 个月；
3. **数据隔离**：举报数据与素材库物理分库；
4. **安全审计**：每季度进行 1 次安全自查，每年聘请第三方进行 1 次安全评估；
5. **应急响应**：发生或可能发生数据泄露时，72 小时内启动应急响应并报告监管部门。

## 五、信息共享、转让与公开披露

### 5.1 共享

本小程序**不向任何第三方共享**未成年人个人信息或使用数据。

### 5.2 转让

本小程序**不向任何第三方转让**用户数据。如因合并、收购等需要转让，将提前 30 天公示，并要求监护人重新同意。

### 5.3 公开披露

除法律法规强制要求或监管部门依法调取外，本小程序**不进行任何形式的公开披露**。

### 5.4 第三方 SDK 清单

本小程序**未接入任何第三方 SDK**（包括但不限于统计 SDK、广告 SDK、推送 SDK、登录 SDK）。所有功能均由开发者自研实现。

## 六、未成年人特别保护条款

### 6.1 适用对象

本条款适用于 2-6 岁儿童及其监护人。本小程序假设所有使用者均为未成年人，采取最高保护标准。

### 6.2 监护人同意机制

1. 首次启动小程序时，**强制**进入"家长验证页"（详见《未成年人保护方案》第 1 章）；
2. 监护人完成简单问答或扫码验证后，方可进入主界面；
3. 进入"成人科普"页面时，**二次**要求监护人确认；
4. 未通过验证的，一律按未成年人模式呈现内容。

### 6.3 内容分级承诺

1. 2-6 岁主界面内容严格遵守"三无"标准：无暴力、无恐怖、无性暗示；
2. 涉及"流血 / 做梦 / 死亡"等敏感话题，仅以"成人科普"层隐藏呈现，需家长验证后查看；
3. 实验类内容强制提示"家长陪同"。

### 6.4 时间管理承诺

1. 单次使用 ≤ 15 分钟，每日累计 ≤ 30 分钟；
2. 22:00-次日 6:00 默认禁用（监护人可在控制面板关闭）；
3. 时间到达后强制退出至家长验证页。

### 6.5 商业化限制

1. 未成年人内容**无任何广告**；
2. **无虚拟支付、无内购**；
3. **无诱导分享 / 关注 / 转发**；
4. **无排行榜 / 无竞技机制**。

### 6.6 不收集承诺

本小程序郑重承诺：不以任何形式收集、存储、使用未成年人个人信息。所有使用数据均存储于本地，监护人可随时清除。

## 七、用户权利

### 7.1 查询权

监护人可通过"家长控制面板 > 数据查看"查询本地存储的所有数据，包括：
- 累计使用时长
- 家长验证记录
- 内容过滤设置
- 举报历史

### 7.2 更正权

监护人可在"家长控制面板"中更正：
- 内容过滤等级
- 使用时段限制
- 通知偏好

### 7.3 删除权

监护人可通过以下方式删除数据：
1. **一键清除本地数据**：家长控制面板 > 重置隐私协议同意；
2. **删除举报记录**：向 `support@billion-whys.example` 发送邮件，注明举报编号，我们将在 15 个工作日内删除并邮件回执；
3. **卸载小程序**：卸载后微信将自动清除本地 storage。

### 7.4 撤回同意权

监护人可随时通过"家长控制面板 > 撤回同意"撤回对本政策的同意，撤回后小程序将无法继续使用，直至监护人重新同意。

### 7.5 账号注销

本小程序不要求用户注册账号，因此不涉及账号注销。本地数据清除等同于"注销"。

### 7.6 响应时限

我们将在收到监护人请求后 **15 个工作日内**予以响应，复杂情况不超过 30 个工作日。

## 九、隐私协议变更通知机制

1. **重大变更**：提前 7 天在小程序首屏弹窗公示，要求监护人重新确认同意；
2. **一般变更**：在"家长控制面板 > 协议历史"中公示，不强制重新同意；
3. **紧急变更**（如法律强制要求）：立即生效，并在 24 小时内通过弹窗通知；
4. **历史版本**：保留 3 年，可向开发者邮箱申请获取。

变更类型判定标准：
- 重大变更：收集范围、使用目的、共享对象、监护人权利实现机制发生变化；
- 一般变更：联系邮箱、文档排版、文字润色等。

## 附录 A：术语定义

- **未成年人**：根据《民法典》第 19、20 条，未满 18 周岁的自然人为未成年人，其中不满 8 周岁为无民事行为能力人。本小程序目标用户为 2-6 岁儿童，属无民事行为能力人，所有处理行为均须由监护人代为作出。
- **监护人**：未成年人父母或依法指定监护人，对本小程序隐私同意、数据清除、举报等行为享有决定权。
- **个人信息**：以电子或其他方式记录的能够单独或者与其他信息结合识别特定自然人身份的各种信息。
- **敏感个人信息**：包括但不限于生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹、不满 14 周岁未成年人个人信息。
