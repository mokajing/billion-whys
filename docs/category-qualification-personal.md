# 「十亿个什么与为什么」微信小程序类目资质申请材料（个人主体版）

> 面向 2-6 岁儿童的好奇心即时响应产品，包含 270 个儿童问题与配套配图。
> 文档目的：在**个人主体**注册前提下，为小程序类目申请、内容提审提供完整的主体与资质决策依据。
> 与 `category-qualification.md`（企业主体版）并列存在，本文件为现行实际执行版本。

---

## 1. 主体类型决策

### 1.1 为什么选择个人主体

经综合评估注册门槛、变现路径、内容审核通过率与项目阶段，本项目首期采用**个人主体**注册小程序。

理由：

1. **快速上线验证**：个人主体仅需身份证即可注册，1-3 个工作日完成，便于尽快跑通 MVP 与内容质量验证。
2. **免费内容定位**：本期定位为"完全免费的好奇心问答工具"，不涉及付费、会员、虚拟商品，个人主体的商业化限制不影响核心产品形态。
3. **广告变现路径可行**：个人主体的小程序在累计访问用户（UV）≥ 1000 后即可申请开通"流量主"，接入 banner 广告与激励视频广告，**个人主体同样可以接入微信广告**。
4. **类目选择有解**：避开教育、亲子、医疗等对个人主体关闭的类目后，"工具-效率""工具-信息查询"对个人主体开放，且资质门槛为零。
5. **后续可平滑迁移**：个人主体跑通后，可在公司注册完成时迁移至企业主体，类目与内容资产可继承。

### 1.2 个人主体的硬性限制（已确认并接受）

| 限制项 | 影响 | 本项目应对 |
| --- | --- | --- |
| 不能开通微信支付 | 无法做会员、订阅、虚拟商品付费 | 本期定位免费，无付费功能 |
| 不能申请教育类目 | 教育-素质教育、学历教育、培训机构等均不可 | 主类目选择工具-效率，规避教育类目 |
| 不能申请亲子-母婴亲子类目下的部分子类 | 涉及商品销售的子类不可申请 | 本期无商品销售，纯内容 |
| 不能申请医疗、金融、社交、网络媒体类目 | 不适用 | 本项目不涉及 |
| 一个身份证最多 2 个小程序 | 名额受限 | 仅占用 1 个名额 |
| 流量主开通需累计 UV ≥ 1000 | 不可立即变现 | 阶段化策略，先增长再变现（详见 ad-monetization-plan.md） |
| 未成年人内容审核更严 | 提审通过率偏低 | 配合 wx-miniprogram-audit-team 技能做 270 题自审 |
| 责任承担为无限责任 | 法律风险个人承担 | 严格遵守内容审核与未成年人保护规范 |

### 1.3 后续迁移至企业主体的触发条件

满足以下任一条件时启动迁移：

- DAU 稳定 ≥ 5000，广告月收入 ≥ ¥3000，需要扩展付费/会员等商业化能力；
- 与出版社、教育机构、IP 方达成合作，对方要求企业资质；
- 监管或微信平台政策变更，个人主体无法继续承载；
- 团队扩张至 5 人以上，需要规范化的股权与劳动关系。

## 2. 类目选择决策

参考微信小程序类目表（https://developers.weixin.qq.com/miniprogram/product/），在个人主体前提下，对候选类目进行分析。

### 2.1 候选类目 A：工具-效率（**首选**）

#### 适配性分析
- 产品核心功能是"问题→答案→配图"的即时响应，本质上是知识查询/问答工具。
- 工具-效率类目强调"提升效率、解决问题"，好奇心问答可包装为"快速回答孩子问题"的效率工具。
- **适配度：中高**。功能层面契合，资质门槛对个人主体友好。

#### 资质要求
- 主体资质：**个人身份证 + 实名认证即可**，无行业专项资质要求。
- 无需 ICP 经营许可证。
- 无需教育部门资质。
- 无需出版社资质。

#### 优缺点
- **优点**：资质门槛最低、审核周期最短、对个人主体开放、提审通过率高。
- **缺点**：类目语义与"儿童内容"距离较远，可能影响用户搜索权重；未成年人保护审查仍需独立通过。

### 2.2 候选类目 B：工具-信息查询（**备选**）

#### 适配性分析
- 产品本质是"问题→答案"的查询行为，与信息查询类目高度契合。
- 配图可视为"信息呈现"的一部分。
- **适配度：高**。功能契合度最高。

#### 资质要求
- 主体资质：**个人身份证 + 实名认证即可**。
- 无行业专项资质。
- 如涉及 UGC（用户提问），需提供内容审核机制说明。本项目仅展示预置 270 题，无 UGC，无此要求。
- 无需教育/出版资质。

#### 优缺点
- **优点**：资质门槛低、审核快、功能契合度最高、对个人主体开放。
- **缺点**：类目语义中性，不突出儿童与亲子属性。

### 2.3 个人主体下不可申请的类目（受限清单）

以下类目**对个人主体关闭**，本项目不会选择：

| 类目 | 受限原因 |
| --- | --- |
| 教育-素质教育 / 学历教育 / 培训机构 | 需企业资质 + 教育部门许可 |
| 亲子-母婴亲子（部分子类） | 涉及商品销售的子类需企业资质 |
| 医疗-互联网医院 / 医疗信息 | 需医疗机构资质 |
| 金融-理财 / 保险 / 银行 | 需金融监管资质 |
| 支付 / 虚拟支付类 | 个人主体不可开通微信支付 |
| 社交-社区 / 论坛 | 需企业资质 + ICP + 内容审核机制认证 |
| 网络媒体-资讯 | 需互联网新闻信息服务许可 |

## 3. 推荐方案

- **首选类目：工具-效率**
  - 理由：对个人主体开放、资质门槛为零、审核周期短、产品功能（即时问答）契合。
- **备选类目：工具-信息查询**
  - 理由：功能契合度更高，若工具-效率审核受阻可作为首选 fallback。
- **不申请：教育、亲子、医疗、金融、支付、社交、网络媒体类目**
  - 理由：均对个人主体关闭或要求企业资质 + 行业许可。

## 4. 资质材料清单（针对推荐类目：工具-效率）

### 4.1 必备材料

- [ ] 个人身份证（正反面彩色扫描件，清晰完整，在有效期内）
- [ ] 个人实名认证（微信扫码绑定本人银行卡或人脸核身）
- [ ] 个人手机号（用于注册管理员）
- [ ] 个人微信号（用于绑定管理员）
- [ ] 邮箱（用于登录账号与接收审核通知）
- [ ] 小程序名称与 Logo
- [ ] 小程序服务协议、隐私政策（合规必备）
- [ ] 未成年人保护方案（专用文档，必备，因目标用户含未成年人）

### 4.2 类目专项材料

工具-效率类目**无需额外行业资质材料**，仅需在类目申请时填写：

- [ ] 类目申请说明（在线填写，说明产品定位、核心功能、目标用户）
- [ ] 内容来源说明（270 题原创声明 / 授权协议）
- [ ] 内容审核机制说明（人工 + 机器审核流程、敏感词词库、未成年人保护策略）

### 4.3 可选材料（提审加分项）

- [ ] 软件著作权登记证书（270 题内容著作权，个人可作为著作权人）
- [ ] ICP 备案（个人主体可办，需服务器与域名，非强制但建议）
- [ ] 内容安全评估报告（自审报告）

## 5. 申请流程

### 5.1 注册个人小程序账号

1. 访问 https://mp.weixin.qq.com/，点击"立即注册"。
2. 选择账号类型：**小程序**。
3. 填写邮箱（作为登录账号，未注册过微信公众平台）。
4. 激活邮箱，设置密码。
5. 选择主体类型：**个人**。
6. 填写个人信息：姓名、身份证号、身份证正反面扫描件。
7. 通过人脸核身或银行卡验证完成实名认证。
8. 填写管理员信息：个人手机号、扫码绑定本人微信号。
9. 等待审核（1-3 个工作日）。
10. **个人主体无需缴纳 300 元/年认证费**（个人主体免费）。

### 5.2 类目申请步骤

1. 登录小程序后台 → 设置 → 基本设置 → 服务类目。
2. 点击"修改"，选择"工具-效率"。
3. 选择子类（如有，按产品实际形态选择；若仅一级类目则直接选择）。
4. 在线填写类目申请说明（产品定位、核心功能、目标用户、内容来源）。
5. 上传类目专项材料（内容来源说明、内容审核机制说明）。
6. 提交审核，等待微信审核员审核（3-7 个工作日）。
7. 审核通过后，类目正式生效，可提交小程序版本审核。

### 5.3 提审前准备

- [ ] 完成小程序基本信息设置（名称、Logo、简介、客服联系方式）
- [ ] 完成《隐私政策》《用户协议》发布
- [ ] 完成《未成年人保护方案》发布（必备，因目标用户含未成年人）
- [ ] 完成 270 题内容自审（敏感词、价值观、未成年人保护，参考 wx-miniprogram-audit-team 技能的 270 题自审清单）
- [ ] 完成配图自审（无低俗、暴力、政治敏感内容）
- [ ] 配置客服（响应时长 ≤ 5 分钟，未成年人内容要求更严）
- [ ] 提交代码审核前完成「未成年人内容合规自检」

### 5.4 提审后跟进

- 关注审核结果通知（邮件 + 站内信）。
- 如被驳回，根据驳回原因整改后重新提交（驳回原因一般在 24 小时内可在后台查看）。
- 重点关注以下高频驳回点：
  - 类目与功能不一致（确保首页功能与工具-效率类目描述一致）
  - 未成年人保护方案不完整
  - 内容含敏感信息
  - 隐私政策未声明收集信息用途
- 审核通过后，发布上线；首次发布后 7 日内重点监控举报与投诉。

## 6. 风险提示

### 6.1 类目与功能不一致风险

- **风险**：若选择工具-效率类目但实际功能被审核员判定为"教育/亲子"性质，可能被驳回或下架。
- **应对**：首页与导航突出"问答工具""快速查询""家长查询孩子问题"的语义；功能描述、Banner、引导文案均使用工具语境；**避免使用"教学""课程""学习""教育"** 等教育类关键词；避免使用"亲子陪伴"等亲子类关键词。

### 6.2 内容版权风险（270 题版权归属）

- **风险**：270 题与配图若未明确版权归属，可能被投诉侵权。
- **应对**：
  - 提前完成 270 题内容的著作权登记（个人可作为著作权人，登记于中国版权保护中心或地方版权局）。
  - 配图如使用 AI 生成，保留生成模型授权许可、提示词记录。
  - 配图如使用外部素材，签署书面授权协议。
  - 在小程序内显著标注版权声明。
  - 建立侵权投诉响应机制（24 小时响应）。

### 6.3 未成年人内容特殊要求

- **风险**：目标用户含 2-6 岁儿童，未成年人保护审查是重点，违反可能导致永久封禁。
- **应对**：
  - 完成未成年人保护方案（必备）：未成年人个人信息保护、内容分级、家长同意机制。
  - 不得收集未成年人精确位置、人脸等敏感信息。
  - **不得向未成年人推送广告**（未成年人模式关闭广告，详见 minor-protection.md 第 5 章与第 8 章）。
  - 内容不得含暴力、恐怖、低俗、不良价值观。
  - 配置未成年人模式（一键切换至纯儿童内容）。

### 6.4 个人主体额外风险

- **风险**：个人主体承担无限责任，如发生重大合规事件（违规内容、数据泄露、监管处罚），个人需承担全部法律后果。
- **应对**：
  - 严格遵守内容审核与未成年人保护规范；
  - 不接入未授权第三方 SDK；
  - 不收集任何儿童个人信息；
  - 配置完整的举报与响应机制；
  - 一旦 DAU 与商业化规模达到阈值（详见 1.3 节），立即迁移至企业主体。

## 7. 时间预估

### 7.1 各阶段时间

| 阶段 | 预估时间 | 备注 |
| --- | --- | --- |
| 个人小程序账号注册 | 1-3 个工作日 | 主体认证审核 |
| 类目申请审核 | 3-7 个工作日 | 工具类目审核较快 |
| 软件著作权登记（如办理） | 30-60 个工作日 | 与类目审核并行 |
| ICP 备案（如办理） | 20 个工作日 | 与类目审核并行，非强制 |
| 内容自审与整改 | 5-7 个工作日 | 270 题与配图逐项审查 |
| 小程序代码审核 | 1-7 个工作日 | 首次审核可能多轮 |
| 发布上线 | 1 个工作日 | 审核通过后即可发布 |

### 7.2 总周期预估

- **最短路径（不办软著/ICP）**：约 2-3 周
  - 注册（3 天）+ 类目审核（5 天）+ 内容自审（5 天）+ 代码审核（3 天）+ 缓冲（3 天）
- **推荐路径（办理软著）**：约 4-6 周
  - 类目审核与软著登记并行，关键路径为软著登记 30-60 天 + 内容自审 7 天 + 代码审核 7 天

### 7.3 关键里程碑

- M1：个人小程序账号注册完成
- M2：类目（工具-效率）申请提交
- M3：类目审核通过
- M4：270 题内容与配图自审完成
- M5：小程序代码审核通过，正式上线
- M6：累计 UV ≥ 1000，开通流量主（详见 ad-monetization-plan.md）
